EDAT Developer API

Integrate real-time toll data, emission indices, and carbon pricing into your smart city applications. Our privacy-first API ensures zero PII leakage — all vehicle references use SHA-256 anonymous hashes.

Quick Start

⧉ Copy
# Get a live toll transaction feed
curl -X GET https://api.edat.gov.my/v2/transactions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  --data '{"limit": 100, "corridor": "E1-N", "include_emission": true}'
Base URL
https://api.edat.gov.my/v2
Authentication
Bearer Token (JWT)

1,000 req/min

Privacy Whitepaper

Technical explanation of how EDAT achieves zero-PII data processing.

SHA-256 Anonymisation Pipeline

1

Camera Capture

OpenCV frame captured at 30fps. License plate region extracted via YOLO detection with ≥85% confidence threshold.

→
2

Edge Hashing

The OCR-read plate string is hashed on the edge device itself before any network transmission. Raw plate never leaves the camera unit.

# Python — runs on-device
import hashlib
hash_id = hashlib.sha256(b"WXY1234").hexdigest()
→
3

Hash Transmitted

Only sha256("WXY1234") = "a3f9c2..." is sent to EDAT servers. No plate, no identity.

→
4

Irrecoverability

SHA-256 is a one-way function. Even EDAT operators cannot reverse the hash to find the original plate. Mathematically guaranteed.

What We Store

  • ✓ SHA-256 hash (64 hex chars)
  • ✓ Vehicle class (EV/ICE/etc.)
  • ✓ Emission index float
  • ✓ Toll amount & timestamp
  • ✓ Corridor & lane ID

What We Never Store

  • ✗ Licence plate number
  • ✗ Owner name or MyKAD
  • ✗ GPS location data
  • ✗ Camera footage (discarded)
  • ✗ Journey patterns or routes
